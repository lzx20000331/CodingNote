# 认识 Ajax

Asynchronous JavaScript + XML（异步 JavaScript 和 XML），其本身不是一种新技术，而是一个在 2005 年被 Jesse James Garrett 提出的新术语，用来描述一种使用现有技术集合的‘新’方法，

应用

1.注册页面的表单验证

2.百度搜索时的下拉框

3.京东网页的一级菜单

4.图片，文章等资源的懒加载

## 优点

在网页不刷新的情况下向服务端进行通信

允许根据用户行为来更新内容

## 缺点

没有浏览历史， 不能回退

存在跨域问题

SEO 不太友好（**SEO** (搜索引擎优化) 是一种让网站在搜索引擎结果中更加清晰, 也帮助我们将搜索结果更靠前）

# 原生 ajax

```js
//1.创建对象
const xhr = new XMLHttpRequest();
//2.初始化,设置请求方式和url
xhr.open("get", url);
//3.发送
xhr.send();
//4.时间绑定,处理服务端返回的结果
xhr.onreadystatechange = function () {
  if (xhr.readyState === 4) {
    if (xhr.status >= 200 && xhr.status < 300) console.log("yes");
    else console.log("no");
  }
};
```

- 服务端

```js
const express = require("express");

const app = express();

app.get("/", (request, response) => {
  //设置允许跨域
  response.header("Access-Control-Allow-Origin", "*");
  //允许的header类型
  response.header("Access-Control-Allow-Headers", "content-type");
  //跨域允许的请求方式
  response.header(
    "Access-Control-Allow-Methods",
    "DELETE,PUT,POST,GET,OPTIONS"
  );
  response.send("hello ajax");
});

app.listen(8080, () => {
  console.log("服务已启动!");
});
```
